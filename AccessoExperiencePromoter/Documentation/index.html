Developer User Guide for the Accesso Experience Promoter SDK for Android

Installing the SDK

  1. Place the AccessoExperiencePromoter-release.aar file and the AccessoCore-release.aar file within the app/libs directory.

  2. Include a dependency to the the SDK in the module-level build.gradle file and add required dependencies

    implementation(files("libs/AccessoExperiencePromoter-release.aar"))
    implementation(files("libs/AccessoCore-release.aar"))
    implementation("com.squareup.retrofit2:retrofit:2.9.0")
    implementation("com.squareup.retrofit2:converter-gson:2.9.0")
    implementation("org.altbeacon:android-beacon-library:2.19.4")

  1. Add required permissions and replace theme.

    <manifest>
        <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
        <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
        <uses-permission android:name="android.permission.INTERNET" />
        <application>
            tools:replace="android:theme"
        </application>
    </manifest>

  1. Sync gradle.

Setup SDK Configurations

  1. Place the accesso-services.json file within the app/src/main/assets directory

  2. The SDK will read the config files on initialization

Initializing the SDK

  • Before requesting messages, initialize the SDK with the required builder functions.

        AccessoCore
            .setAppUserId(appUserId)
            .setApplicationContext(applicationContext)
            .setActivity(activity)
            .build()
Content copied to clipboard

  • appUserId is the UUID of the device

  • applicationContext is the context of the host application

  • activity is the activity that initializes the SDK

  • build() must be called after setting configurations

  • Create an instance of AccessoExperiencePromoter

    val accessoExperiencePromoter = AccessoExperiencePromoter()

Optional Builder Functions

  • To begin detecting beacons, add the setOnBeaconRegionDetection builder function.

        AccessoCore
            .setAppUserId(appUserId)
            .setApplicationContext(applicationContext)
            .setActivity(activity)
            .setOnBeaconRegionDetection(onBeaconRegionDetection)
            .build()
Content copied to clipboard

  • onBeaconRegionDetection is a callback function to get notified of enter/exit events in a given region.

Retrieving Messages

  • After AccessoCore has been configured, you can begin calling methods needed for your application, such as getMessages(venueId).

  • venueId is an optional parameter to get messages associated with that venueId

  • Note AccessoExperiencePromoter.getMessages is a suspend function so it must be called within a kotlin coroutine

        coroutineScope.launch {
            try {
                val messages = AccessoExperiencePromoter.getMessages(venueId)
            } catch (e: Exception) {
                
            }
        }
Content copied to clipboard

  • AccessoExperiencePromoter.getMessages() returns a Response<List<MessageResponse?>>

Enable Push Notifications

  • Setup a push notification service of your choice (such as Firebase)

  • Prompt the user to enable push notifications through your application

  • Once a notification is received call again getAllMessages() to refresh the list of messages.

Start Listening for Beacons Given the Beacon Regions

  • After fetching the beacon regions, if you would like to start monitoring specific regions, this is how you can call the listen function.

  • This function must not be called with GlobalScope.

     val beaconListener: (BeaconRegion, Int) -> Unit = { beaconRegion, state ->
        if (state == EnterExit.ENTER.ordinal) {
            Log.d(
                "SampleAppBeacons",
                "Entered beacon range for beaconRegion=${beaconRegion}"
            )
        } else {
            Log.d(
                "SampleAppBeacons",
                "Exited beacon range for beaconRegion=${beaconRegion}"
            )
        }
    }

     coroutineScope.launch {
          AccessoExperiencePromoter.startListeningForBeaconRegions(beaconRegions, beaconListener)
     }

Stop Listening for Beacons Given the Beacon Regions

  • After startListeningForBeaconRegions is called, you can call stopListeningForBeaconRegions at anytime with the given beaconRegions to stop listening to them. If you provide no regions to the call, it will stop listening to all beacon regions.

     coroutineScope.launch {
          AccessoExperiencePromoter.stopListeningForBeaconRegions()
     }
